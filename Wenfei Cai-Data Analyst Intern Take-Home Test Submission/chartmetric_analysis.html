Introduction This notebook provides SQL solutions for the take-home case study. Each SQL block is followed by an explanation to illustrate the business logic.

Q1: Retrieve artist names with total number of tracks (sorted descending)

SELECT ca.name AS artist_name, COUNT(DISTINCT l.cm_track) AS total_tracks FROM cm_artist ca JOIN l_cm_track_cm_artist l ON ca.id = l.cm_artist GROUP BY ca.name ORDER BY total_tracks DESC;

Explanation: Joins artist and linking table to count how many tracks each artist is associated with. Groups by artist name and sorts descending by total track count.

Q2: Retrieve artist names with total tracks and earliest album release year

SELECT ca.name AS artist_name, COUNT(DISTINCT l1.cm_track) AS total_tracks, MIN(EXTRACT(YEAR FROM a.release_date)) AS earliest_album_year FROM cm_artist ca JOIN l_cm_track_cm_artist l1 ON ca.id = l1.cm_artist JOIN l_cm_track_cm_album l2 ON l1.cm_track = l2.cm_track JOIN cm_album a ON l2.cm_album = a.id GROUP BY ca.name ORDER BY ca.name;

Explanation: Combines artist-track-album relationships. Retrieves both track count and earliest album release year for each artist.

Q3: Find track names associated with multiple artists

SELECT t.name AS track_name, STRING_AGG(DISTINCT a.name, ', ') AS artist_names FROM cm_track t JOIN l_cm_track_cm_artist l ON t.id = l.cm_track JOIN cm_artist a ON l.cm_artist = a.id GROUP BY t.name HAVING COUNT(DISTINCT a.id) > 1;

Explanation: Joins tracks and artists, groups by track name. Only returns tracks linked to more than one artist.

Bonus Q4: Average & Running Total of Follower Count per Artist

SELECT a.name AS artist_name, AVG(c.follower_count) OVER (PARTITION BY a.id) AS avg_follower_count, SUM(c.follower_count) OVER (PARTITION BY a.id ORDER BY c.created_at) AS running_total FROM cm_artist a JOIN cm_artist_cache_history c ON a.id = c.cm_artist;

Explanation: Uses window functions to calculate average follower count and running total for each artist. Assumes 'created_at' shows snapshot order for follower history.

Correlation Between YouTube Views and Spotify Plays

I analyzed the correlation between YouTube views and Spotify plays. After removing the top 1% outliers, the Pearson correlation was around 0.72, suggesting a strong linear relationship. However, the data was heavily skewed.

To address this, I applied a log transformation to both variables. This reduced skewness and highlighted the underlying trend. The correlation remained strong (r ≈ 0.70), and the log-log scatter plot showed a much clearer linear pattern.

This transformation helps reveal a more reliable and interpretable relationship, especially across platforms with different usage scales.

import pandas as pd
from sklearn.preprocessing import MinMaxScaler

# Step 1: Select the indicators to be used for scoring
weights = {
    # Market influence (35%)
    'YOUTUBE_VIEWS': 0.07,
    'SPOTIFY_PLAYS': 0.07,
    'TIKTOK_TOP_VIDEOS_VIEWS': 0.07,
    'GENIUS_PAGEVIEWS': 0.07,
    'SHAZAM_COUNT': 0.07,
    # Activity level (25%)
    'TIKTOK_POSTS_MONTHLY_DIFF_PER': 0.05,
    'SHAZAM_COUNT_MONTHLY_DIFF_PER': 0.05,
    'YOUTUBE_VIEWS_MONTHLY_DIFF_PER': 0.05,
    'SOUNDCLOUD_PLAYS_MONTHLY_DIFF_PER': 0.05,
    'SPOTIFY_PLAYS_MONTHLY_DIFF_PER': 0.05,
    # Business value (25%)
    'PANDORA_LIFETIME_STREAMS': 0.08,
    'AIRPLAY_STREAMS': 0.06,
    'SIRIUSXM_STREAMS': 0.06,
    'SOUNDCLOUD_PLAYS': 0.05,
    # Professional recognition (15%)
    'CM_TRACK_RANK': 0.08,
    'TIDAL_POPULARITY': 0.07
}

# Step 2: Read and aggregate data (aggregate average values by artist)

rank_df = df.groupby('ARTIST_NAME')[list(weights.keys())].mean().reset_index()

# Step 3: normalization
scaler = MinMaxScaler()
scaled_values = scaler.fit_transform(rank_df.drop(columns=['ARTIST_NAME']))
scaled_df = pd.DataFrame(scaled_values, columns=rank_df.columns[1:])
scaled_df['ARTIST_NAME'] = rank_df['ARTIST_NAME'].values

# Step 4: Calculate the total score (sum up according to the weighted formula)
scaled_df['total_score'] = sum(scaled_df[k] * w for k, w in weights.items())

# Step 5: Ranking by quantile classification
quantiles = scaled_df['total_score'].quantile([0.95, 0.85, 0.65])

def assign_rank(score):
    if score >= quantiles[0.95]:
        return 'S'
    elif score >= quantiles[0.85]:
        return 'A'
    elif score >= quantiles[0.65]:
        return 'B'
    else:
        return 'C'

scaled_df['RANK'] = scaled_df['total_score'].apply(assign_rank)

# Step 6: Map the order of the levels and sort them, retaining the final result
rank_order = {'S': 1, 'A': 2, 'B': 3, 'C': 4}
scaled_df['rank_order'] = scaled_df['RANK'].map(rank_order)

Artist Performance Scoring System Overview

This scoring system evaluates artists based on their aggregated performance across major music platforms. It is designed to be multi-dimensional, weighted, explainable, and easily adjustable.

  1. Dimensions & Metric Selection

I divide artist performance into four key dimensions, each with carefully selected indicators:

  • Market Influence (35%)

    • YOUTUBE_VIEWS, SPOTIFY_PLAYS, TIKTOK_TOP_VIDEOS_VIEWS, GENIUS_PAGEVIEWS, SHAZAM_COUNT

  • Activity & Momentum (25%)

    • Monthly growth indicators such as:
      TIKTOK_POSTS_MONTHLY_DIFF_PER, YOUTUBE_VIEWS_MONTHLY_DIFF_PER, SPOTIFY_PLAYS_MONTHLY_DIFF_PER, etc.

  • Commercial Value (25%)

    • AIRPLAY_STREAMS, PANDORA_LIFETIME_STREAMS, SIRIUSXM_STREAMS, SOUNDCLOUD_PLAYS

  • Professional Recognition (15%)

    • CM_TRACK_RANK, TIDAL_POPULARITY
  1. Data Processing & Standardization
  • Track-level data is grouped by ARTIST_NAME and averaged to obtain artist-level metrics.
  • A MinMaxScaler is applied to normalize all features, ensuring comparability across different scales.
  • A weighted sum based on the assigned weights is calculated to derive a single total_score per artist.
  1. Ranking & Tiering Logic

  • Artists are ranked based on quantiles of their total_score:

    • Top 1% → S Tier
    • 95–99% → A Tier
    • 80–95% → B Tier
    • Below 80% → C Tier

  • An ordinal map is applied to sort by rank first, then by total score in descending order.

  1. Use Cases

This scoring system is suitable for:

  • Internal resource allocation (e.g., promotional priorities)
  • Talent scouting and partnership evaluation
  • Executive dashboards and performance tracking (e.g., in Looker or Tableau)

The model is flexible and adaptable—weights and dimensions can be fine-tuned based on business goals or data insights.
